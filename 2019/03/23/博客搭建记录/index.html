博客搭建记录

一、环境搭建，软件安装

1.安装node.js

下载安装(如图)

node.js包含两个组件:npm和node.js本身
检验node.js安装完成:打开命令行，分别node -v和npm -v(如图)

2.git的下载与安装

官网下载
安装教程参考:https://blog.csdn.net/ainuser/article/details/77609180(利用此方法安装后貌似会自动配置环境变量)

3.hexo安装(使用淘宝npm镜像)

任意位置打开Git Bash
输入命令:$ npm install -g cnpm --registry=https://registry.npm.taobao.org
出现warn 忽略
输入:$ cnpm install hexo --save
安装完成后输入命令检测:$ hexo -v
(坑1:使用"$ cnpm install hexo --save"安装后hexo -v出现"bash: hexo: command not found
"换用cnpm install -g hexo-cli即可)
成功如图


二、hexo本地生成博客

1.创建文件夹

命名随意(我这里是新建了个blog文件夹)

2.进入该文件夹 右键运行git bash

3.hexo初始化:$ hexo init

(坑2:一定要新建文件夹，当初为了方便直接在安装hexo目录下初始化，结果报错找不到文件)
 成功后查看blog文件夹内容(如图)

4.生成博客:hexo g(如图)

5.本地部署测试

输入以下命令，把生成的博客部署到本地，并测试访问
$ hexo s
本地访问 http://localhost:4000/&emsp

(如图)

(注意:在git bash界面，不能用ctrl+C复制，只能右键copy)

三、github服务器部署

1.注册github账号

2.新建仓库 命名规则username.github.io

3.部署配置 hexo本地的配置文件_config.yml中关联上刚才在github上新创建的仓库:

hexo本地文件夹根目录下_config.yml文件 配置并保存
deploy:
type: git
repo: git@github.com:github账户名/github账户名.github.io.git
branch: master
(如图)

(注意空格，一定不能少)

4.安装hexo发布插件

用于把本地生成的博客，通过上面配置的地址，发布到github上的仓库中
还是在hexo源文件夹下打开git bash,输入以下命令:
$ npm install hexo-deployer-git –save

5.配置SSH key

输入:$ ssh-keygen -t rsa -C “你的github注册邮件地址”
然后连续3次回车，最终会生成一个.ssh\id_rsa.pub文件
打开你的github主页，进入仓库，进入个人设置 -> SSH and GPG keys -> New SSH key：添加该ssh key ：title随便取，key就填刚刚生成的文件全文
测试是否添加成功:$ ssh -T git@github.com
(成功如图)

(坑3:之前由于安装hexo插件不同版本间的问题导致hexo d报错”ERROR Deployer not found: git”根据提示输入”$ npm audit fix”解决，卸载hexo:npm uninstall hexo-cli -g)
6.博客上传
$ hexo clean 清理
$ hexo g(generate) 生成
$ hexo d(deploy) 部署
然后在浏览器中输入http://yourgithubname.github.io就可以看到你的个人博客了

四、域名绑定

1.购买域名(我是万网购买 解析之前需要实名)
2.设置域名解析
点击添加解析，记录类型选A或CNAME，A记录的记录值就是ip地址，github(官方文档)提供了两个IP地址，192.30.252.153和192.30.252.154，这两个IP地址为github的服务器地址，两个都要填上，解析记录设置两个www和@，线路就默认就行了，CNAME记录值填你的github博客网址。如: username.github.io。
(坑4:github pages支持https且ip地址过时，解析好后邮箱会受到邮件警告，先用命令行ping自己的用户名.github.io得到的ip换入即可)
hexo根目录的source文件夹里创建CNAME文件，不带任何后缀，里面添加你的域名信息 如：name.com
(github仓库里进入设置，下拉至github pages 在custom domain添加域名，且github仓库页面刷新后应该会多出一些文件)
